# 一、项目介绍
  ## 目的
  - 帮助普通用户，在自定义设置的时间段，自动调度脚本以完成安卓游戏里的日常任务，同时提供多设备管理、任务编排、日志查看、邮件通知功能
  - 帮助开发者用户方便、快捷地开发、调试脚本运行逻辑
  ## 主要技术
  ### 前端
  - Tauri + vue3 + taiwindcss + vite + lucide-vue
  - 其余内容按需查看package.json
  ### 后端
  - image + ort + adb_client + interprocess + rhai
  - 其余内容按需查看cargo.toml
  ## 架构
  - Tauri调度+配置+查看，子进程运行脚本+模型推理
# 二、功能优化/新增/重构时需注意的内容
  ## 总体
  - 单个文件行数尽可能短
  - 此次改动，是否涉及外部模块或包？优先考虑本地已安装内容
  ## 前端
  - style改动，项目里其它地方是否做过相同样式改动？若有，则尽可能复用
  - 能否抽离成组件？抽离原则：复用或精简或方便维护
  - 此次改动，是否影响后端？影响时同步修改前端
  - 此次改动，ts对象是否是自动生成？是的话重新运行生成命令
  - ts函数里，对数据的判断，该数据是否是结构化内容？
    - 是，该内容是否能直接从自动生成的ts对象里取？
      - 能，从ts对象获取，保持数据来源一致
      - 不能，历史代码里该对象是否已写？是的话在原来对象里添加
    - 不是，该内容是否是常量？尽量把常量字符串、工具类函数，放在一份文件里以方便后续维护
  ## 后端
  - 独立：能否将本内容抽离成单独的包或模块？抽离原则：提升开发期间编译速度或功能复用或精简或方便维护
  - 新增结构体：新增结构体时考虑该内容是否已存在相同或相似内容？若有，则复用，没有才新增
  - 新增函数：优先保证UI响应。即除开保证数据一致性的情况外，其余内容尽可能异步调用，不阻塞UI响应
