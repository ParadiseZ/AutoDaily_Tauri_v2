1. 将所有 `[待填写]` 替换为实际运营主体名称、统一社会信用代码（如有）、联系邮箱、联系地址、服务器所在地及生效日期；在客户端注册、登录、赞助、上传和下载页面放置可访问链接。
2. 注册前新增“我已阅读并同意《用户协议》《隐私政策》”的**非预勾选**复选框，并记录版本号、时间、账户及 IP/设备日志；重要变更时重新取得同意。
3. 最低限度在注册页设置“我已满 14 周岁；否则已取得监护人同意”的非预勾选声明，未确认不得注册，并记录版本与时间；收到儿童使用线索时暂停账户并处理信息。如改为主动面向儿童提供服务，须先取得可验证的监护人同意并落实《儿童个人信息保护规则》。仅放文档不够。
4. 客户端兑换前须展示《兑换码权益与客服规则》：兑换码对应的权益、期限、适用范围、失效/冻结条件、客服和申诉渠道。兑换码销售页须由实际销售主体展示价格、退款、发票/收据与售后规则；不要将有对价的权益称为“无对价捐赠”。
5. 云端脚本上传须有作者声明和举报/下架入口：上传者保证其脚本、模型、图像、文本和链接具有合法权利，且不含游戏账号、身份证件、支付信息、未成年人信息或恶意代码。上线前落实审核、投诉处理与留痕。
6. 核实生产后端实际收集的 IP、访问日志、兑换码、验证码、备份、客服记录及保存期限，并逐项补入隐私政策和第三方共享清单；客户端代码不能证明后端未收集这些信息。若客户端、后端或兑换系统将账户标识发送给售码方，须将该共享关系单列披露。

8. 用构建产物生成完整 SBOM 与第三方开源许可证归档（Rust、npm、模型、图标、字体及安装器），与版本号一起发布；当前 `Cargo.lock`/`package.json` 不能替代许可证文本。

## 还需落实的产品边界

- AutoDaily 是本地 Android 自动化工具：本地编辑、运行、批量运行和示例脚本免费，不提供游戏、不代表任何游戏开发者/发行商；兑换码仅限制他人公开分享的云端脚本的下载、运行资格，用于覆盖云端成本。使用自动化可能违反第三方服务规则并导致封号或其他损失，须在首次运行及脚本市场显著提示。
- 云端脚本的下载、运行资格可与有效兑换码权益绑定，但不得暗示权益能保证功能可用、脚本结果、账号安全或第三方游戏许可。
- 若使用境外 GitHub 更新/配置服务或境外服务器，先完成跨境个人信息评估并在清单中披露；未完成前改用合规的境内服务或关闭相关能力。
