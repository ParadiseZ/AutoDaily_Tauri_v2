# 前端用户视角审查记录

日期：2026-04-29

## 已直接处理的前端展示问题

- 关于页：去掉内部实现导向的描述，补充设备、脚本、运行三个用户可理解的信息块，避免首屏左侧大卡片过空。
- 脚本市场：运行时筛选补回“全部运行时”，避免下拉框只有单一选项导致用户误以为必须筛选 Rhai。
- 脚本编辑器：给结构操作工具条增加标题，减少标题区下方整条空白工具带的突兀感。
- 视觉测试：收敛窗口头部和侧栏说明文案，改为展示当前选择与数量状态，降低说明文字占屏。

## 需要审查后再做的功能类改动

- 脚本市场空态目前只能提示“没有找到匹配脚本”。如果要展示推荐脚本、最近更新、分类、同步失败原因或离线状态，需要市场接口提供对应数据和错误分类。
- 运行日志空态目前无法区分“确实没有日志”和“日志监听/读取失败”。如果需要更明确的用户反馈，需要后端或 IPC 暴露读取状态与错误原因。
