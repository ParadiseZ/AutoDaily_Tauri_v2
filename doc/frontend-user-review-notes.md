# 前端用户视角审查记录

日期：2026-04-29

## 已直接处理的前端展示问题

- 全局视觉：去掉页面背景的装饰性光斑，降低共享面板的圆角、阴影、控件高度和空态高度，让主界面更接近高频使用的桌面工具。
- 本地列表：左侧脚本卡片增加数量摘要和本地/云端标记；迁移脚本不再直接展示 `Migrated from old Android server...`，改成可读摘要。
- 本地列表详情：把“打开编辑器/编辑信息”和“上传/克隆/清理/删除”分层，危险操作不再和主要操作挤在同一排。
- 关于页：去掉内部实现导向的描述，补充设备、脚本、运行三个用户可理解的信息块，避免首屏左侧大卡片过空。
- 运行日志：无日志时不再显示一整块黑色终端，改为普通空态，并根据是否有筛选条件提示“没有日志”或“筛选后没有日志”。
- 脚本市场：运行时筛选补回“全部运行时”，避免下拉框只有单一选项导致用户误以为必须筛选 Rhai。
- 脚本市场：空态补充下一步提示，并标明推荐/分类/失败原因需要云端接口状态支撑。
- 脚本编辑器：收紧顶部标题区，改中文标签、缩短保存按钮文案，并给结构操作工具条增加标题，减少标题区下方整条空白工具带的突兀感。
- 视觉测试：收敛窗口头部和侧栏说明文案，改为展示当前选择与数量状态，降低说明文字占屏。

## 需要审查后再做的功能类改动

- 脚本市场如果要展示推荐脚本、最近更新、分类、同步失败原因或离线状态，需要市场接口提供对应数据和错误分类。
- 运行日志如果要区分“还没有运行日志”“日志监听未连接”“读取失败”，需要后端或 IPC 暴露监听状态与错误原因。
- 关于页如果要升级成诊断页，需要后端提供数据目录、日志目录、数据库路径、当前 Tauri/WebView 版本、配置文件路径等只读状态。
